# AegisMonitor 本地端到端演示步骤

## 1. 演示目标

本步骤用于课程设计答辩前自检，验证以下主链路可以在本机跑通：

1. Spring Boot 后端连接本地 MySQL。
2. Agent 首次注册并获得身份。
3. Agent 上报心跳、主机指标和服务发现结果。
4. 运维大屏所需接口可以查询主机、最新指标、服务清单和告警。

## 2. 前置条件

- JDK 17 已安装。
- Maven 已可用，或使用本项目 `backend/test.cmd` 中配置的 Maven。
- MySQL Server 已启动，并存在 `aegis_monitor` 数据库。
- Python 已可用，Agent 依赖已安装 `psutil`。
- `backend/application-local.yml` 已从 `backend/application-local.example.yml` 复制并填写本机 MySQL 密码。

注意：`backend/application-local.yml` 和 Agent 状态文件都不应提交到 Git。

## 3. 后端启动

在项目根目录执行：

```powershell
cd backend
mvn spring-boot:run
```

若当前终端没有 Maven，可使用已安装的 Maven 绝对路径启动：

```powershell
cd backend
C:\Users\QiXing\dev-tools\apache-maven-3.9.16\bin\mvn.cmd spring-boot:run
```

启动成功后，后端默认监听：

```text
http://localhost:8080
```

## 4. Agent 配置

复制示例配置：

```powershell
copy agent\agent.example.yml agent\agent.yml
```

默认示例会连接本机后端：

```text
server_url: http://localhost:8080/api
register_token: demo-register-token
host_alias: demo-host-a
```

如果要模拟多台主机，可为不同机器修改 `host_alias`，并删除本机已有的 `agent\.agent-state.json` 后重新运行。

## 5. Agent 单次上报

在项目根目录执行：

```powershell
agent\run-once.cmd --config agent\agent.yml
```

如果 `python` 不在 PATH 中，先指定 Python 解释器：

```powershell
$env:AEGIS_AGENT_PYTHON="C:\Users\QiXing\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe"
agent\run-once.cmd --config agent\agent.yml
```

成功时输出类似：

```text
run-once completed: agentId=agt_001 hostId=host_001
```

## 6. 接口自检

查询主机列表：

```powershell
Invoke-RestMethod http://localhost:8080/api/agents
```

查询最新主机指标：

```powershell
Invoke-RestMethod "http://localhost:8080/api/metrics/host/latest?hostId=host_001"
```

查询服务清单：

```powershell
Invoke-RestMethod "http://localhost:8080/api/services?hostId=host_001"
```

查询告警列表：

```powershell
Invoke-RestMethod http://localhost:8080/api/alerts
```

初始化前端演示数据：

```powershell
Invoke-RestMethod `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"hostCount":3,"includeServices":true,"includeAlerts":true}' `
  http://localhost:8080/api/demo/seed
```

预期效果：

- `GET /api/agents` 可看到 3 台模拟主机；如果再运行真实 Agent，可组合展示更多主机。
- `GET /api/services?hostId=demo_host_001` 可看到 NGINX 服务实例。
- `GET /api/alerts` 可看到一条 `demo_host_003` 的 CPU 高危告警，用于演示 ACK 闭环。

## 7. 前端答辩点击路线

启动前端：

```powershell
cd frontend
npm.cmd run dev
```

默认访问：

```text
http://127.0.0.1:5173/hosts
```

推荐展示顺序：

1. 进入“主机监控”，点击“初始化演示数据”，观察主机总数、在线主机、模拟主机和待处理告警。
2. 在主机列表点击 `demo-web-01` 或真实 Agent 主机，进入主机详情，展示 CPU、内存和 TCP 最新快照。
3. 点击侧边栏“服务组件”，选择 `demo-web-01`，展示 NGINX 服务和端口发现结果。
4. 点击侧边栏“告警中心”，找到 `demo_host_003` 的 CPU 高危告警。
5. 点击 `ACK`，填写确认人和处理备注，提交后观察状态变为 `ACKED`。

说明：前端“初始化演示数据”按钮只在开发模式或显式演示模式显示；如果使用生产构建演示，可先通过本节接口自检中的 `POST /api/demo/seed` 命令初始化数据。

## 8. 常见处理

- 如果 Agent 重复运行没有重新注册，这是正常行为；它会复用 `agent\.agent-state.json`。
- 如果需要重新演示首次注册，删除 `agent\.agent-state.json` 后再运行。
- 如果端口 `8080` 被占用，先关闭占用进程，或临时修改后端启动端口和 Agent `server_url`。
- 如果 MySQL 连接失败，优先检查 `backend/application-local.yml` 是否存在、密码是否正确、`aegis_monitor` 数据库是否已创建。
- 如果提示缺少 `psutil`，请对 `AEGIS_AGENT_PYTHON` 指向的 Python 执行 `python -m pip install psutil`。

## 9. 答辩讲解点

- Agent 采用主动上报，避免后端逐台扫描主机。
- 注册后使用 `agentId + agentSecret` 做后续请求凭证。
- 指标、服务清单、告警是三条可展示的运维主线。
- 告警 ACK 体现真实运维闭环。
