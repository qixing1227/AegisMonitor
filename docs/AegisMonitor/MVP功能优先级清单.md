# AegisMonitor MVP 功能优先级清单

## 1. MVP 判断标准

AegisMonitor 的 MVP 不是功能最多的版本，而是能在课程答辩中完整证明“一体化监控平台”核心价值的最小版本。

MVP 必须证明：

1. Agent 能接入。
2. 指标能采集。
3. 数据能存储。
4. 前端能展示。
5. 告警能触发和处理。
6. 系统能用模拟数据保证演示稳定。

## 2. P0：必须完成

P0 任务不完成，系统主链路无法成立。

| 功能 | 所属模块 | 说明 |
| --- | --- | --- |
| Agent 本地配置读取 | Agent | 读取后端地址、Token、采集周期 |
| Agent 注册 | Agent / 后端 | Token 校验后创建主机和 Agent 记录 |
| Agent 心跳 | Agent / 后端 | 支撑在线状态和离线判断 |
| 主机基础信息采集 | Agent | 主机名、系统、CPU 核心、内存总量 |
| CPU 指标采集 | Agent | 总使用率、每核使用率 |
| 内存指标采集 | Agent | 总量、已用、可用、使用率 |
| 磁盘指标采集 | Agent | 分区容量和使用率 |
| 网络指标采集 | Agent | 网卡收发字节和速率 |
| TCP 指标采集 | Agent | TCP 连接数和监听端口 |
| 指标上报接口 | 后端 | 接收 Agent 指标 |
| InfluxDB 写入 | 后端 / 数据库 | 存储高频指标数据 |
| MySQL 基础表 | 后端 / 数据库 | 存储用户、主机、Agent、服务、告警 |
| 主机列表接口 | 后端 | 前端查询主机资产 |
| 主机详情接口 | 后端 | 前端查询主机信息和指标 |
| Vue 基础项目 | 前端 | 前端运行基础 |
| 主机列表页面 | 前端 | 展示接入主机 |
| 主机详情页面 | 前端 | 展示指标曲线 |
| Docker Compose 数据库 | 部署 | 启动 MySQL 和 InfluxDB |

## 3. P1：核心展示能力

P1 任务影响答辩质量，应在 P0 稳定后尽快完成。

| 功能 | 所属模块 | 说明 |
| --- | --- | --- |
| 登录与 JWT | 后端 / 前端 | 支撑权限与企业化展示 |
| 基础 RBAC | 后端 / 前端 | 实现超级管理员和运维工程师 |
| 服务识别 | Agent / 后端 | 识别 Java、MySQL、Redis、Nginx、Node.js |
| 服务组件页面 | 前端 | 展示服务实例和进程指标 |
| 告警规则管理 | 后端 / 前端 | CPU、内存、磁盘、离线、服务停止 |
| 告警事件生成 | 后端 | 超阈值后生成告警 |
| 告警中心 | 前端 | 展示和筛选告警 |
| 告警 ACK 与关闭 | 后端 / 前端 | 体现运维闭环 |
| 模拟数据生成 | 后端 / 数据 | 生成 3 台模拟主机和指标 |
| Swagger UI | 后端 | 展示接口文档 |
| 单元测试 | 测试 | 覆盖告警、权限、Agent 识别 |
| 接口测试 | 测试 | 覆盖主要 REST API |
| 集成测试 | 测试 | 验证完整链路 |
| 答辩演示脚本 | 文档 | 保证展示顺序稳定 |

## 4. P2：加分能力

P2 任务不是 MVP 生死线，但会显著提升系统完整度。

| 功能 | 所属模块 | 说明 |
| --- | --- | --- |
| Agent 纳管页面 | 前端 / 后端 | 展示 Token、接入主机、最近心跳 |
| 用户管理基础页面 | 前端 / 后端 | 超级管理员管理用户 |
| 服务拓扑接口 | 后端 | 返回节点和调用关系 |
| 服务拓扑页面 | 前端 | 展示 Nginx -> Spring Boot -> MySQL / Redis |
| 模拟调用链详情 | 后端 / 前端 | 展示耗时、状态、时间点 |
| 主机分组与标签筛选 | 前端 / 后端 | 区分生产、测试、演示和模拟数据 |
| 异常测试记录 | 测试 | Token 错误、Agent 离线、空数据、数据库不可用 |

## 5. P3：只写入展望

P3 不进入本次实现范围，避免拖垮项目。

| 功能 | 原因 |
| --- | --- |
| Linux Agent | 当前演示环境以 Windows 为主 |
| Agent 远程配置下发 | 需要配置版本、拉取机制、失败回滚 |
| Agent 自动升级 | 涉及安装包、版本兼容和安全校验 |
| WebSocket / SSE | 定时轮询已满足演示 |
| 邮件/短信/企业微信通知 | 需要外部通道和账号配置 |
| OpenTelemetry 真实 Trace | 超出课程 MVP 范围 |
| Kubernetes / Docker 指标 | 需要额外运行环境 |
| 多租户隔离 | 课程项目无需覆盖 |
| 复杂告警收敛 | 实现成本高，MVP 用基础去重即可 |

## 6. 最小可演示闭环

如果时间非常紧，必须优先保证以下路径：

1. 启动 MySQL 和 InfluxDB。
2. 启动 Spring Boot 后端。
3. 启动 Vue 前端。
4. 启动 Windows Agent。
5. Agent 注册成功。
6. Agent 上报主机指标。
7. 前端主机列表看到主机在线。
8. 主机详情看到 CPU、内存、磁盘、网络趋势。
9. 生成或触发一条告警。
10. 在告警中心确认并关闭告警。

这条链路跑通后，AegisMonitor 就已经具备课程答辩意义上的核心价值。
