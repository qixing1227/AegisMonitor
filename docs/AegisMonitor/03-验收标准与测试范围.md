# AegisMonitor 验收标准与测试范围

## 1. 验收目标

本项目验收目标是证明 AegisMonitor 已实现一体化监控平台的核心闭环：

Agent 采集数据，后端接收与存储，前端展示监控状态，系统根据规则触发告警，运维人员处理告警，并通过服务拓扑辅助定位问题。

## 2. MVP 功能验收清单

| 编号 | 功能 | 验收方式 | 通过标准 |
| --- | --- | --- | --- |
| AC-001 | 用户登录 | 前端登录页面 + 接口测试 | 正确账号可登录，错误账号被拒绝 |
| AC-002 | 角色权限 | 前端菜单 + 后端接口 | 超级管理员和运维工程师权限不同 |
| AC-003 | Agent 注册 | 启动 Agent | 后端生成主机记录，前端可看到新主机 |
| AC-004 | Agent 心跳 | 等待心跳上报 | 主机状态显示在线，停止 Agent 后可变为离线 |
| AC-005 | 主机指标 | 查看主机详情 | CPU、内存、磁盘、网络、TCP 指标可展示 |
| AC-006 | 指标入库 | 查询后端或数据库 | 指标写入 InfluxDB，资产写入 MySQL |
| AC-007 | 服务识别 | 启动典型服务或使用模拟数据 | 能展示 Java/Spring Boot、MySQL、Redis、Nginx、Node.js 中的服务 |
| AC-008 | 告警触发 | 调低阈值或使用模拟告警 | 超阈值后生成告警事件 |

| AC-014 | Swagger 文档 | 浏览后端接口文档 | 可查看主要 REST API |

## 3. 测试范围

### 3.1 单元测试

重点对象：

- Agent 主机指标转换逻辑。
- Agent 服务识别逻辑。
- 后端告警规则判断逻辑。
- 后端权限判断逻辑。
- 后端 DTO 参数校验。

示例用例：

| 编号 | 测试对象 | 输入 | 预期输出 |
| --- | --- | --- | --- |
| UT-001 | CPU 阈值判断 | CPU 使用率 90%，阈值 80% | 生成 CRITICAL 或 WARNING 告警 |
| UT-002 | Token 校验 | 错误 Agent Token | 注册失败 |
| UT-003 | 服务识别 | 进程名 mysqld.exe，端口 3306 | 识别为 MySQL |
| UT-004 | 角色权限 | OPS_ENGINEER 访问用户管理接口 | 被拒绝 |

### 3.2 接口测试

重点接口：

- 用户登录接口。
- Agent 注册接口。
- Agent 心跳接口。
- 主机指标上报接口。
- 服务发现上报接口。
- 主机查询接口。
- 指标查询接口。
- 告警规则接口。
- 告警确认接口。

示例用例：

| 编号 | 接口 | 场景 | 预期 |
| --- | --- | --- | --- |
| API-001 | POST /api/auth/login | 正确账号密码 | 返回 JWT |
| API-002 | POST /api/agents/register | 正确 Token | 返回注册成功 |
| API-003 | POST /api/agents/register | 错误 Token | 返回 401 或业务错误 |
| API-004 | POST /api/metrics/host | 合法指标数据 | 写入成功 |
| API-005 | GET /api/hosts | 登录后查询 | 返回主机列表 |
| API-006 | POST /api/alerts/{id}/ack | 确认告警 | 告警状态变为已确认 |

### 3.3 集成测试

重点链路：

1. Agent 启动。
2. Agent 注册到后端。
3. Agent 上报主机指标。
4. 后端写入 MySQL 和 InfluxDB。
5. 前端刷新主机详情图表。
6. 调低阈值触发告警。
7. 运维工程师确认并关闭告警。

通过标准：

- 链路中无接口异常。
- 前端数据和后端数据一致。
- 告警状态流转正确。

### 3.4 异常测试

重点场景：

- Agent Token 错误。
- Agent 停止运行。
- 后端不可达。
- InfluxDB 不可用。
- MySQL 不可用。
- 上报数据字段缺失。
- 指标为空时前端图表展示。
- 重复注册同一主机。
- CPU、内存、磁盘阈值被设置为异常值。

## 4. 答辩演示验收路径

建议演示顺序：

1. 展示项目背景和整体架构。
2. 启动 Docker Compose 中的 MySQL 和 InfluxDB。
3. 启动 Spring Boot 后端并打开 Swagger UI。
4. 启动 Vue 前端并登录系统。
5. 启动 Windows Agent，展示主机自动接入。
6. 查看主机详情，展示 CPU、内存、磁盘、网络、TCP 指标。
7. 展示服务组件识别结果。
8. 展示 3 台模拟主机，体现多主机场景。
9. 调低 CPU 或内存阈值，触发告警。
10. 在告警中心执行 ACK 和关闭。
11. 展示服务拓扑和模拟调用链。
12. 展示测试文档和课程设计报告。

## 5. 验收风险与兜底方案

| 风险 | 兜底方案 |
| --- | --- |
| 真实 Agent 接入失败 | 使用模拟主机数据继续演示 |
| 指标变化不明显 | 将采集频率调为 3 秒或使用模拟指标 |
| 告警触发失败 | 使用预置告警事件或调低阈值 |
| 数据库启动失败 | 提前准备截图和本地导出数据 |
| 第二台主机无法联网 | 在同一台电脑运行第二个 Agent 配置或使用模拟主机 |
