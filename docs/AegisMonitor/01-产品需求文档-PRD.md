# AegisMonitor 一体化监控平台 PRD

## 1. 背景

随着企业数字化转型推进，IT 系统逐步上云，系统架构从单体应用演进为多主机、多组件、多服务协同的复杂形态。主机数量增长、PaaS 组件增多、应用微服务化，使运维人员需要同时关注主机资源、服务状态、组件指标和告警事件。

本项目选择“中国电信广东公司实训课题 2026 年版”中的课题二：一体化监控平台开发。系统通过部署 Agent 自动采集主机与服务信息，并通过服务端和监控台帮助运维人员快速发现、定位和处理问题。

## 2. 产品定位

AegisMonitor 是一个面向课程设计场景的一体化监控平台原型。它以可运行演示为核心目标，以生产级监控平台的架构思想为设计参考，实现从 Agent 采集、数据上报、指标存储、监控展示到告警处理的完整闭环。

## 3. 目标

### 3.1 业务目标

- 帮助运维人员集中查看多台主机的运行状态。
- 自动识别主机上的典型服务和技术栈。
- 展示 CPU、内存、磁盘、网络、TCP 连接等关键指标。
- 支持阈值告警和告警确认闭环。
- 通过服务拓扑和模拟调用链辅助定位问题。
- 支持真实 Agent 数据和模拟数据并存，保证课程答辩演示稳定。

### 3.2 工程目标

- 按软件工程流程产出可行性分析、项目计划、需求说明、设计说明、测试说明和课程设计报告。
- 建立前后端分离、Agent 独立采集、双数据库存储的系统结构。
- 使用 Swagger/OpenAPI 形成接口文档。
- 使用单元测试、接口测试、集成测试和异常测试验证核心链路。

## 4. 非目标

- 不建设企业生产级监控平台。
- 不支持大规模主机集群。
- 不实现 Agent 自动升级和远程配置下发。
- 不实现完整分布式链路追踪。
- 不接入真实短信、邮件、企业微信告警通知。
- 不实现复杂多租户和完整数据隔离。
- 不采集 Kubernetes、Docker、云厂商资源等扩展指标。

## 5. 用户角色

### 5.1 超级管理员 System Admin

负责平台整体配置和生命周期管理。

主要权限：

- 用户管理和角色分配。
- 生成 Agent 注册 Token。
- 查看和纳管接入主机。
- 管理全局配置。
- 查看全部监控数据和告警事件。

MVP 实现范围：

- 登录。
- 用户管理基础能力。
- Agent Token 或注册配置管理。
- 查看全部主机和告警。

### 5.2 运维工程师 Ops Engineer

负责日常监控和告警处理，是系统的主要使用者。

主要权限：

- 查看监控仪表盘。
- 查看主机列表和主机详情。
- 查看服务组件和服务拓扑。
- 修改阈值告警规则。
- 确认和关闭告警，并填写处理备注。

MVP 实现范围：

- 登录。
- 查看监控数据。
- 管理告警规则。
- 处理告警事件。

### 5.3 研发工程师 / 业务方 Developer Read-only

负责查看与自己应用相关的只读监控面板。

主要权限：

- 只读查看被授权主机或应用的监控数据。
- 只读查看应用层指标。
- 不能修改告警规则。
- 不能删除主机或服务。

MVP 实现范围：

- 不做真实实现。
- 在设计文档中保留角色定义，体现最小权限原则和数据隔离扩展方向。

## 6. 核心场景

### 6.1 Agent 接入主机

超级管理员生成 Agent 注册 Token。运维人员在 Windows 主机上配置 Agent 的服务端地址、Token、采集频率和主机别名。Agent 启动后调用注册接口，后端校验 Token 并建立主机资产记录。注册成功后，Agent 定期上报心跳、主机指标和服务识别结果。

### 6.2 查看主机运行状态

运维工程师登录监控台，在总览仪表盘查看在线主机数、告警数量、CPU/内存整体利用率和最近指标趋势。进入主机详情页后，可以查看基础信息、CPU、内存、磁盘、网络和 TCP 连接数据。

### 6.3 自动识别服务组件

Agent 定期扫描进程、端口和命令行信息，识别 Java/Spring Boot、MySQL、Redis、Nginx、Node.js 等典型服务。服务端接收识别结果后更新服务实例列表，前端展示服务名称、技术栈类型、PID、端口、运行状态和进程指标。

### 6.4 告警触发与确认

系统根据阈值规则判断 CPU、内存、磁盘、Agent 离线和服务停止等异常情况。当触发告警时，后端写入告警事件。运维工程师在告警中心查看告警详情，点击确认告警，填写处理备注，并在处理完成后关闭告警。

### 6.5 服务拓扑与模拟调用链

系统根据服务识别结果和演示配置展示轻量服务拓扑。例如 Nginx 调用 Spring Boot API，Spring Boot 访问 MySQL 和 Redis。点击拓扑节点或调用边后，前端展示模拟调用耗时、状态码、时间点和链路状态。

### 6.6 答辩演示模式

系统支持真实主机和模拟主机并存。答辩时先展示真实 Agent 接入，再展示 3 台模拟主机的多节点监控效果。模拟数据包含最近 30 分钟指标、典型服务实例和告警事件。

## 7. 功能范围

### 7.1 Agent 模块

- 读取本地配置文件。
- 使用注册 Token 注册主机。
- 定时发送心跳。
- 采集主机基础信息。
- 采集 CPU、内存、磁盘、网络和 TCP 连接指标。
- 扫描并识别典型服务。
- 上报服务进程指标。
- 处理服务端不可达、Token 错误等异常。

### 7.2 后端服务模块

- 用户登录和 JWT 签发。
- RBAC 权限校验。
- Agent 注册和 Token 校验。
- 主机资产管理。
- 服务实例管理。
- 指标数据写入 InfluxDB。
- 资产、配置、告警、用户数据写入 MySQL。
- 告警规则管理。
- 告警事件生成、确认和关闭。
- 模拟数据生成。
- Swagger/OpenAPI 接口文档。

### 7.3 前端监控台模块

- 登录页。
- 总览仪表盘。
- 主机列表。
- 主机详情。
- 服务组件页面。
- 服务拓扑页面。
- 告警中心。
- 告警规则管理。
- Agent 管理。
- 用户管理基础页面。

### 7.4 数据存储模块

MySQL 存储：

- 用户与角色。
- Agent 注册信息。
- 主机资产。
- 服务实例。
- 告警规则。
- 告警事件。
- 系统配置。

InfluxDB 存储：

- CPU 指标。
- 内存指标。
- 磁盘指标。
- 网络指标。
- TCP 连接指标。
- 服务进程指标。

## 8. MVP 验收标准

| 编号 | 验收项 | 验收标准 |
| --- | --- | --- |
| AC-001 | Agent 注册 | Windows Agent 可使用 Token 注册到后端 |
| AC-002 | Agent 心跳 | 后端能判断 Agent 在线状态 |
| AC-003 | 主机指标采集 | 前端可查看 CPU、内存、磁盘、网络、TCP 指标 |
| AC-004 | 指标存储 | 高频指标写入 InfluxDB，资产和告警写入 MySQL |
| AC-005 | 服务识别 | 至少识别 Java/Spring Boot、MySQL、Redis、Nginx、Node.js 中的若干典型服务 |
| AC-006 | 监控台展示 | 前端能展示仪表盘、主机列表、主机详情、服务组件和告警中心 |
| AC-007 | 告警规则 | CPU、内存、磁盘、Agent 离线、服务停止可触发告警 |
| AC-008 | 告警闭环 | 运维工程师可确认和关闭告警 |
| AC-009 | 权限控制 | 超级管理员和运维工程师具有不同菜单或接口权限 |
| AC-010 | 演示模式 | 系统可生成模拟主机、指标、服务和告警数据 |
| AC-011 | API 文档 | Swagger UI 可查看主要接口 |
| AC-012 | 测试文档 | 提供单元、接口、集成和异常测试记录 |

## 10. 主要风险

| 风险 | 影响 | 应对策略 |
| --- | --- | --- |
| InfluxDB 部署失败 | 指标无法存储和查询 | 使用 Docker Compose，并保留模拟数据兜底 |
| 真实主机连接不稳定 | 答辩演示中断 | 准备 3 台模拟主机和演示数据 |
| 服务识别不准确 | 展示效果下降 | 先支持进程名和端口识别，文档说明后续增强 |
| 告警触发不明显 | 演示难以体现闭环 | 提供可调阈值和模拟告警数据 |
| 功能范围过大 | 项目延期 | 严格遵守 MVP，不做超范围能力 |
