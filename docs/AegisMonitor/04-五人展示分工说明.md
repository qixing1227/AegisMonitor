# AegisMonitor 五人展示分工说明

## 1. 分工原则

虽然项目实际开发可以由少数人集中完成，但课程答辩时需要体现团队协作和软件工程分工。AegisMonitor 的展示分工按照系统模块、文档成果和答辩内容进行划分，保证每名成员都有清晰、可讲、可展示的负责范围。

## 2. 角色 1：项目经理 / 系统架构师

### 角色定位

负责项目统筹、需求边界、技术选型和系统架构，是整个项目的总负责人。

### 负责内容

- 可行性分析。
- 项目计划。
- 需求规格说明书统筹。
- 系统总体架构设计。
- 技术选型说明。
- API 约定和模块边界。
- 答辩流程统筹。

### 展示时可讲内容

- 为什么选择课程设计级可运行系统，而不是生产级系统。
- 为什么采用 Agent + Spring Boot + Vue + MySQL + InfluxDB 架构。
- 文档化瀑布 + 敏捷迭代的过程模型。
- 系统整体数据流。

## 3. 角色 2：Agent 开发工程师

### 角色定位

负责被监控主机侧的数据采集和服务识别。

### 负责内容

- Windows Agent。
- 本地配置文件。
- Agent 注册 Token。
- 心跳上报。
- 主机基础信息采集。
- CPU、内存、磁盘、网络、TCP 指标采集。
- Java/Spring Boot、MySQL、Redis、Nginx、Node.js 服务识别。

### 展示时可讲内容

- Agent 为什么主动 HTTP 上报。
- psutil 如何采集主机指标。
- 如何通过进程名、端口和命令行识别服务。
- 采集频率为什么这样设计。

## 4. 角色 3：后端开发工程师

### 角色定位

负责系统核心业务逻辑、数据存储、权限认证和告警处理。

### 负责内容

- Spring Boot 后端。
- MyBatis-Plus 访问 MySQL。
- InfluxDB Java Client 写入和查询指标。
- JWT 用户认证。
- Agent Token 认证。
- 主机资产管理。
- 服务实例管理。
- 告警规则和告警事件。
- Swagger/OpenAPI 接口文档。

### 展示时可讲内容

- 为什么 MySQL 和 InfluxDB 分离。
- 后端模块化单体如何划分。
- 告警事件如何从指标数据中生成。
- 用户 JWT 和 Agent Token 为什么分开。

## 5. 角色 4：前端开发工程师

### 角色定位

负责监控台界面、图表展示和运维交互。

### 负责内容

- Vue 3 + Vite 前端项目。
- Element Plus 后台管理界面。
- ECharts 监控图表。
- 登录页。
- 总览仪表盘。
- 主机列表和主机详情。
- 服务组件页面。
- 服务拓扑和模拟调用链。
- 告警中心。
- Agent 管理和用户管理基础页面。

### 展示时可讲内容

- 为什么采用后台管理台为主、Dashboard 带大屏感的风格。
- 指标图表如何定时刷新。
- 告警确认闭环如何在界面上体现。
- 服务拓扑如何辅助定位问题。

## 6. 角色 5：测试与文档工程师

### 角色定位

负责测试设计、缺陷验证、文档整理和最终交付质量。

### 负责内容

- 测试计划。
- 单元测试用例。
- 接口测试用例。
- 集成测试用例。
- 异常测试用例。
- 测试说明书。
- 课程设计报告。
- 答辩演示脚本。
- 截图和文档排版。

### 展示时可讲内容

- 测试为什么分为单元、接口、集成和异常测试。
- Agent 离线、Token 错误、阈值超限等异常如何验证。
- 如何保证答辩现场有模拟数据兜底。
- 最终交付物有哪些。

## 7. 展示分工与系统模块对应表

| 成员角色 | 对应系统模块 | 对应文档 |
| --- | --- | --- |
| 项目经理 / 系统架构师 | 总体架构、模块边界、项目计划 | 可行性分析、项目计划、概要设计、课程设计报告 |
| Agent 开发工程师 | Windows Agent、指标采集、服务发现 | 详细设计 Agent 部分、测试说明 Agent 部分 |
| 后端开发工程师 | Spring Boot、MySQL、InfluxDB、认证、告警 | 概要设计、详细设计、接口文档 |
| 前端开发工程师 | Vue 监控台、图表、告警中心、拓扑 | 详细设计前端部分、演示脚本 |
| 测试与文档工程师 | 测试、截图、报告、排版 | 测试说明书、课程设计报告、答辩材料 |
