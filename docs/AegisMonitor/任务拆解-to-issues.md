# AegisMonitor 任务拆解

## 1. 拆解原则

本任务拆解从 PRD 和需求规格说明书出发，将 AegisMonitor 拆成可执行、可验证、可分工展示的工程任务。每个任务都尽量满足以下条件：

- 有明确输入和输出。
- 有清晰验收标准。
- 能归属到具体模块或角色。
- 能映射到 Sprint。
- 不包含 MVP 明确不做的超范围能力。

## 2. 优先级定义

| 优先级 | 含义 |
| --- | --- |
| P0 | MVP 主链路必须完成，不完成则系统无法演示 |
| P1 | 核心展示能力，影响答辩质量 |
| P2 | 加分能力，可在主链路稳定后实现 |
| P3 | 文档扩展或未来演进，不进入本次编码范围 |

## 3. Sprint 划分

| Sprint | 主题 | 目标 |
| --- | --- | --- |
| Sprint 0 | 需求与准备 | 完成文档、任务拆解、环境和项目结构准备 |
| Sprint 1 | 主链路打通 | Agent 注册、心跳、指标采集、后端接收、数据库写入 |
| Sprint 2 | 监控展示 | 前端展示主机、指标、服务组件和基础仪表盘 |
| Sprint 3 | 告警与权限 | 登录、RBAC、告警规则、告警事件、ACK 闭环 |
| Sprint 4 | 拓扑、测试与答辩 | 服务拓扑、模拟调用链、测试文档、课程报告和演示脚本 |

## 5. Sprint 1：主链路打通

- 模块：Agent
- 优先级：P0
- 负责人角色：Agent 开发工程师
- 依赖：ISSUE-0003

任务内容：

- 定义 `agent.yml`。
- 读取后端地址、注册 Token、主机别名、采集频率。
- 支持配置缺失时给出明确错误。

验收标准：

- Agent 启动时能正确读取配置。
- 配置错误时不会静默失败。

### ISSUE-1002 实现 Agent 注册接口调用

- 模块：Agent
- 优先级：P0
- 负责人角色：Agent 开发工程师
- 依赖：ISSUE-1001、ISSUE-1101

任务内容：

- Agent 启动后调用后端注册接口。

验收标准：

- 正确 Token 可注册成功。
- 错误 Token 注册失败并输出错误信息。

### ISSUE-1003 实现主机基础信息采集

- 模块：Agent
- 优先级：P0

任务内容：

- 采集主机名、操作系统、系统版本、CPU 核心数、内存总量、启动时间。
- 将采集结果转换成后端 DTO 需要的 JSON。

验收标准：

- 采集结果字段完整。
- Windows 环境下数据真实可读。

### ISSUE-1004 实现 CPU、内存、磁盘、网络、TCP 指标采集

- 模块：Agent
- 优先级：P0
- 负责人角色：Agent 开发工程师
- 依赖：ISSUE-1001

任务内容：

- 使用 Python 采集 CPU 总使用率和每核使用率。
- 采集内存总量、已用、可用、使用率。
- 采集磁盘分区容量和使用率。
- 采集网卡收发字节数和速率。
- 采集 TCP 连接数量和监听端口。

验收标准：

- 指标默认每 5 秒采集一次。
- 采集数据格式可被后端接收。
- 单机运行不出现明显性能问题。

### ISSUE-1005 实现 Agent 心跳上报

- 负责人角色：Agent 开发工程师

验收标准：

- 后端可记录最近心跳时间。
