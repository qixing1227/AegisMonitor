# AegisMonitor 课程设计文档体系规划

## 1. 目的

本文档用于规划 AegisMonitor 在软件工程课程设计中的正式交付文档。课程要求强调可行性分析、项目计划、需求分析、概要设计、详细设计、编码、测试和课程设计报告，因此本项目将按照文档化瀑布流程组织材料，并在实际开发中采用敏捷迭代推进。

## 2. 已完成的需求阶段文档

| 文档 | 用途 |
| --- | --- |
| 00-项目决策记录.md | 记录需求澄清阶段确认的关键架构和范围决策 |
| 01-产品需求文档-PRD.md | 描述产品背景、用户角色、核心场景、功能范围和 MVP 验收标准 |
| 02-需求规格说明书.md | 以软件工程格式定义功能需求、非功能需求、接口需求和数据需求 |
| 03-验收标准与测试范围.md | 明确 MVP 验收清单、测试范围和答辩演示路径 |
| 04-五人展示分工说明.md | 将系统模块映射到 5 人小组答辩分工 |

## 5. 建议推进顺序

1. 已完成：确认 PRD 和需求规格。
2. 已完成：拆分 issue 和 Sprint 任务。
3. 已完成：产出概要设计、接口原型、数据模型原型和页面原型。
4. 下一阶段：进入详细设计或 TDD，先打通 Agent 到后端到数据库的主链路。
5. 展示阶段：完善前端、告警、拓扑和模拟数据。
6. 收尾阶段：补全测试说明书、课程设计报告和答辩演示脚本。
